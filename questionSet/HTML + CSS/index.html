    HTML + CSS 基础

    src 和 href 的区别

    src 和 href 都是用来引用外部的资源。

    src

    src 是 source 的简写 src 会将其指向的资源下载并应用到文档内，嵌入当前标签所在位置。当前浏览器解析到src时，会暂停其他资源的下载和处理，直到该资源加载完毕。常用在img、iframe、script标签上。

    href

    href 是 hypertext reference 的简写 href 是超文本引用，它指向一些网络资源，用于当前文档或元素和指向的资源建立关联。当浏览器识别到它指向的⽂件时，就会并⾏下载资源，不会停⽌对当前⽂档的处理。 常用在a、link等标签上。

    元素的 alt 和 title 属性有什么区别

    alt 和 title 同时设置的时候，alt 作为图片的替代文字出现，title 是图片的解释文字

    使用 link 和 @import 导入样式的区别

    1. link是XHTML标签，除了加载CSS外，还可以定义RSS等其他事务；而@import属于CSS范畴，只能加载CSS；
    2. link引用CSS时，在页面载入时同时加载；而@import需要页面网页完全载入以后加载。
    3. link是XHTML标签，无兼容问题；而@import是在CSS2.1提出的，低版本的浏览器不支持。

    script 标签中 async 和 defer 属性有什么用，它们的区别

    scirpt 标签

    defer 和 async属性都是去异步加载外部的JS脚本文件，它们都不会阻塞页面的解析。
    如果没有defer或async属性，浏览器会立即加载并执行相应的脚本。它不会等待后续加载的文档元素，读取到就会开始加载和执行，这样就阻塞了后续文档的加载。

    区别：

    • 执行顺序

      多个带async属性的标签，不能保证加载的顺序；多个带defer属性的标签，按照加载顺序执行；

    • 脚本是否并行执行

      async属性，表示后续文档的加载和执行与js脚本的加载和执行是并行进行的，即异步执行；
      defer属性，加载后续文档的过程和js脚本的加载(此时仅加载不执行)是并行进行的(异步)，js脚本需要等到文档所有元素解析完成之后才执行，DOMContentLoaded事件触发执行之前。

    如果存在async, 脚本会被并行请求，并尽快解析和执行。 defer 这个布尔属性被设定用来通知浏览器该脚本将在文档完成解析后，触发 DOMContentLoaded (en-US) 事件前执行。

    块级元素和行内元素的区别

    • 块级元素

      • 单个元素会独占一行；
      • 元素的宽高、以及内外边距都可设置；
      • 元素在不设置宽度的情况下，会撑满父元素

    • 行内元素

      • 单个元素不会占据一行，多个元素会在一行排列；
      • 元素的宽高不可设置；
      • 内边距可以设置、外边距水平方向有效，竖直方向无效；
      • 元素在不设置宽度的情况下，随内部元素的内容变化

    CSS 长度单位

    mdn css 长度

    单位可分为三类：

    • 相对长度单位，代表以其他的距离作为单位的一种尺寸。
    • 字体相对长度单位，根据一个字符的大小来定义长度的。
    • 绝对长度单位，代表一个物体度量单位。

    常用的 CSS 单位：

    • em，表示元素的font-size值，如果它用在font-size属性上，则它表示元素继承的font-size值。
    • 1h，使用元素的 line-height 属性的计算值。
    • rem，代表根元素的 font-size 大小。
    • vh，可视窗口高度的 1%。
    • vw，可视窗口宽度的 1%。
    • vmin，可视窗口宽度和高度两者之间的最小值。
    • vmax，可视窗口宽度和高度两者之间的最大值。
    • px，对于普通屏幕，通常是一个设备像素，对于高分辨率屏幕，一个css像素往往占多个设备像素。一般来说，每英寸的像素的数量保持在 96 左右， 1px = 1in 的 96 分之一

    简述 rem 及其转换原理

    rem 是 CSS3 新增的相对长度单位，是指相对于根元素 html 的 font-size 计算值的大小。

    默认根元素的 font-size 都是 16px的。如果想要设置 12px 的字体大小也就是 12px/16px = 0.75rem。

    • 由于 px 是相对固定单位，字号大小直接被定死，无法随着浏览器进行缩放；
    • rem 直接相对于根元素 html，避开层级关系，移动端新型浏览器对其支持较好；

    简述伪类和伪元素

    伪类

    CSS 伪类是添加到选择器的关键字，指定要选择的元素的特殊状态。例如，:hover 可被用于在用户将鼠标悬停在按钮上时改变按钮的颜色。

    1
2
3
4
    /* 所有用户指针悬停的按钮 */
button:hover {
  color: blue;
}

    常用的伪类有

    • :hover，用于用户使用指示设备虚指一个元素（没有激活它）的情况，比如鼠标悬停
    • :active，匹配被用户激活的元素，常用在 <a> 和 <button>元素中
    • :focus，表示获得焦点的元素（如表单输入）
    • :link，选中所有尚未访问的链接
    • :visited，表示用户已访问过的链接
    • :checked，表示任何处于选中状态的radio(), checkbox () 或("select") 元素中的option HTML元素("option")
    • :not，用来匹配不符合一组选择器的元素
    • :first-child，表示在一组兄弟元素中的第一个元素
    • :last-child，表示在一组兄弟元素中的最后一个元素

    伪元素

    伪元素是一个附加至选择器末的关键词，允许你对被选择元素的特定部分修改样式。下例中的 ::first-line 伪元素可改变段落首行文字的样式。

    1
2
3
4
5
    /* 每一个 <p> 元素的第一行。 */
p::first-line {
  color: blue;
  text-transform: uppercase;
}

    常用的伪元素有

    • ::before，在选中元素的顶部创建一个伪元素
    • ::after，在选中元素的尾部创建一个伪元素
    • ::selection，应用于文档中被用户高亮的部分，例如鼠标选中文本
    • ::first-letter，选中块级元素第一行第一个字母
    • ::first-line，选中块级元素第一行应用样式

    区别

    • 伪类使用单冒号，伪元素使用双冒号
    • 伪类的操作对象是文档树中已存在的元素，而伪元素是创建一个文档树外的元素

    opacity: 0、visibility: hidden、display: none 优劣和适用场景

    • display: none 不占空间，不能点击，会引起回流
    • visibility: hidden 占据空间，不能点击，引起重绘，子元素可设置 visible 进行显示
    • opacity: 0 占据空间，可以点击，引起重绘，子元素不影响

    CSS 盒模型

    盒模型指的是将页面上的元素表示为一个个矩形的盒子，css决定这些盒子的大小，位置和属性（颜色、背景等）。

    每个盒子的组成为：

    • content: 容纳真实的内容，例如文本，图片
    • padding: 内边距，扩展到内容区域，填充内容区域与边框的间距。
    • border: 容纳边框的区域。
    • margin: 外边距，用空白区域扩展至边框区域，用以分开相邻的元素。

    css中的盒模型包括IE盒模型和标准的W3C盒模型。

    两者区别：

    在标准的盒模型中width指的content部分的宽度，在IE盒模型中，width表示content+左右padding+左右border这三部分的宽度，这使得在计算整个盒子宽度时存在差异。

    标准盒模型的盒子宽度 = 左右border+左右padding+width

    IE模型的盒子宽度 = width

    在CSS3中引入了box-sizing属性, 这个属性定义了浏览器应该如何计算一个元素的总宽度和总高度。

    1
2
3
4
    /* 表示标准盒模型 */
box-sizing: content-box; 
/* 表示IE盒模型 */
box-sizing: border-box;

    浏览器地址栏输入 URL 到页面展示到底发生了什么

    mdn 渲染页面：浏览器的工作原理

    用户在地址栏输入 URL 提交后会进行以下操作：

    1. DNS 查找，寻找页面资源的位置，拿到服务器的 IP 地址
    2. TCP 握手，一旦获取到服务器IP地址，浏览器就会通过TCP”三次握手“与服务器建立连接，让两端尝试进行通信
    3. TLS 协商，在进行真实的数据传输之前需要TLS协商，验证服务器，建立安全的连接
    4. 发送请求，在建立安全连接后，浏览器就代表用户发送一个初始的HTTP GET请求
    5. 解析数据
      • 构建 DOM 树
      • 构建 CSSOM 树
      • 将 DOM 和 CSSOM 组合成一个 Render 树，Render 树保存所有具有内容和计算样式的可见节点
      • 构建渲染树后，开始布局，确定渲染树中所有节点的大小和位置
      • 将各个节点绘制到屏幕上

    说一下 BFC 及其应用

    mdn BFC

    BFC（Block Format Context）块级格式化上下文，是页面盒模型中的一种 CSS 渲染模式，相当于一个独立的容器，里面的元素和外部的元素相互不影响。

    BFC 主要的作用

    • 清除浮动，主要是为了解决，父元素因为子级元素浮动引起的内部高度为0的问题。
    • 防止同一 BFC 容器中的相邻元素间的外边距重叠问题

    创建 BFC 的方式

    • 根元素（<html>）
    • 浮动元素（元素的 float 不是 none）
    • 绝对定位元素（元素的 position 为 absolute 或 fixed）
    • 行内块元素（元素的 display 为 inline-block）
    • 表格单元格（元素的 display 为 table-cell，HTML表格单元格默认为该值）
    • 表格标题（元素的 display 为 table-caption，HTML表格标题默认为该值）
    • 匿名表格单元格元素（元素的 display 为 table、table-row、 table-row-group、table-header-group、table-footer-group（分别是HTML table、row、tbody、thead、tfoot 的默认属性）或 inline-table）
    • overflow 计算值(Computed)不为 visible 的块元素
    • display 值为 flow-root 的元素
    • contain 值为 layout、content 或 paint 的元素
    • 弹性元素（display 为 flex 或 inline-flex 元素的直接子元素）
    • 网格元素（display 为 grid 或 inline-grid 元素的直接子元素）
    • 多列容器（元素的 column-count 或 column-width (en-US) 不为 auto，包括 column-count 为 1）

    如何避免两个元素的 margin 重叠

    mdn 外边距重叠

    margin 是元素向外延伸的边距，只和自身有关。上下元素的下上外边距有时会重叠，实际空出的空间长度变为两外边距中的较长值。

    两个相邻 div 元素的 margin 值都设为20px，它们之间的间距是 20px，此时 margin 出现了重叠，如何避免这种情况？

    解决方法

    • 设置后一个 div 样式

      • display: inline-block
      • display: table-caption
      • float: left
      • position: absolute
      • position: fixed

    • 设置父元素样式

      • display: grid
      • display: inline-grid
      • display: flex；flex-direction: column

    web 页面渲染事件 ✖️

    说一下事件流

    事件是用户或浏览器执行的某种动作，例如单击、鼠标滑过，响应某个事件的函数叫事件处理程序，或事件监听器。
    JS 与 HTML 的交互是用事件实现的。事件流描述了页面接收事件的顺序。

    事件流包括三个阶段

    1. 事件捕获阶段： 从上到下，层层传递，直到目标接收
    2. 处于目标阶段： 确认目标，进行处理
    3. 事件冒泡阶段： 处理完成，往上传递

    addEventListener 方法可以指定事件处理程序的操作，这个方法接收以下参数：

    1. type：监听的事件类型， 例如click
    2. listener：处理事件的函数

    3. options: 可选参数对象

      1. capture：boolean，true表示在捕获阶段调用处理函数，false表示在冒泡阶段调用，IE只支持事件冒泡
      2. once: boolean, 最多调用一次listener, 调用完后会自动移除

    4. useCapture: 可选，如果没有指定，useCapture 默认为 false 。

    说一下重绘和回流

    重绘

    当页面元素样式改变不影响元素在文档流中的位置时（如background-color，border-color，visibility），浏览器只会将新样式赋予元素并进行重新绘制操作。

    回流

    当render tree中元素的尺寸、布局等改变时，浏览器重新渲染DOM的过程。回流也被称为重排，其实从字面上来看，重排更容易让人形象易懂（即重新排版整个页面）。

    什么情况下会触发重绘或重排，如何优化

    触发重绘

    元素样式发生变化，比如 color, background等属性发生变化时会触发重绘。

    触发回流

    改变浏览器窗口的大小，使用一些JavaScript方法，包括计算样式，对DOM进行元素的添加或删除，或是改变元素的class等。回流必定会发生重绘，重绘不一定会引发回流。

    具体操作：

    • 添加或者删除可见的DOM元素；
    • 元素位置改变；
    • 元素尺寸改变——边距、填充、边框、宽度和高度；
    • 内容变化，比如用户在input框中输入文字，文本或者图片大小改变而引起的计算值宽度和高度改变；
    • 页面渲染初始化；
    • 浏览器窗口尺寸改变，resize事件发生时；
    • 计算offsetWidth和offsetHeight属性；
    • 设置style属性的值；

    浏览器优化

    现代浏览器大多是通过队列机制来批量更新布局，浏览器会把修改操作放在队列中，至少一个浏览器刷新（即16.6ms）才会清空队列，但当你获取布局信息的时候，队列中可能会有影响这些属性或方法返回值的操作，即使没有，浏览器也会强制清空队列，触发回流和重绘来确保返回正确的值。

    例如 offsetTop、clientTop、scrollTop、getComputedStyle()、width、height、getBoundingClientRect()，应避免频繁使用这些属性，他们都会强制渲染刷新队列。

    减少重绘和回流

    1. CSS

      • 使用 transform 代替 top；
      • 使用 visibility 替换 display: none，前者引起重绘，后者引发回流；
      • 避免使用 table 布局；
      • 尽可能在 DOM 树的最末端改变 class；
      • 避免设置多层内联样式，CSS 选择符从右往左匹配查找，避免节点层级过多；
      • 将动画效果应用到 position 属性为 absolute 或 fixed 的元素上，避免影响其他元素的布局；
      • 避免使用 CSS 表达式，可能会引发回流；
      • CSS 硬件加速；

    2. Javascript

      • 避免频繁操作样式，修改 class 最好；
      • 避免频繁操作 DOM，合并多次修改为一次；
      • 避免频繁读取会引发回流/重绘的属性，将结果缓存；
      • 对具有复杂动画的元素使用绝对定位，使它脱离文档流；
